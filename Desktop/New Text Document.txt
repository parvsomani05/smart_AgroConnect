Q2. Do as Directed:
A1. Options used in Telnet option negotiation: (Will/Won’t)
A2. Care-of-address is given by Foreign Agent in Mobile IP: (Yes)
A3. Block size used in TFTP: (512 bytes)
B1. Multiplicative Decrease & Slow Start:
* Multiplicative Decrease: Decreases congestion window size by half upon detecting network congestion.
* Slow Start: Gradually increases the congestion window size at the start of a transmission or after a timeout.
OR
B2. Content Format in MIME:
* MIME (Multipurpose Internet Mail Extensions) formats include text/plain, image/jpeg, application/pdf, etc., enabling email attachments and content type specifications.

Q3. Answer Briefly:
A1. SMTP Protocol:
SMTP (Simple Mail Transfer Protocol) facilitates email transmission by following a sequence:
1. Sender establishes a connection with the mail server.
2. Authentication takes place.
3. Mail commands are issued (MAIL FROM, RCPT TO, DATA).
4. Message is transmitted and connection closes.
B1. GET vs. POST Methods in HTTP:
* GET: Sends data via URL query string. Suitable for idempotent operations.
* POST: Sends data in the request body, ideal for sensitive or large payloads.
OR
A2. Characteristics of Mobile IP and Two Crossing Problem:
* Mobile IP Features: Seamless connectivity, handoffs, and routing.
* Two Crossing Problem: Packets traverse through foreign and home agents multiple times, causing inefficiency.
B2. IPSec Authentication Header:
IPSec provides integrity and authentication using an AH header containing security parameters, sequence numbers, and authentication data.

Q4. State Machine Descriptions:
1. TCP State Machine:
Represents states like LISTEN, SYN-SENT, ESTABLISHED, FIN-WAIT, CLOSE-WAIT during a connection lifecycle.
2. DHCP State Machine:
Involves states such as INIT, SELECTING, REQUESTING, BOUND, RENEWING, and REBINDING for IP allocation.

Q5. Format Descriptions:
1. TCP Segment Format:
Includes fields like source port, destination port, sequence number, acknowledgment number, data offset, flags (SYN, ACK), and checksum.
2. DHCP Message Format:
Comprises operation code (op), hardware type, transaction ID, client IP, server IP, and options.
